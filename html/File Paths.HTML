HTML File Paths

A file path describes the location of a file in a web site's folder structure. File paths are used when linking to external files, like:

  • Web pages (<a href="...">)
  • Images (<img src="...">)
  • Style sheets (<link rel="stylesheet" href="...">)
  • JavaScript files (<script src="...">)

Let's assume the following folder structure for our examples:

root/
├── index.html
├── contact.html
├── images/
│ └── picture.jpg
└── pages/
    └── about.html

Absolute File Paths

An absolute file path is the full URL to a file, including the protocol (http:// or https://) and the domain name.

Example:

This links to an image on an external website. This is the only way to link to resources on other domains.

<img src="https://www.w3schools.com/images/picture.jpg" alt="W3Schools logo">

Relative File Paths

A relative file path points to a file relative to the current page. It does not include the domain name.

1. Same Folder

If the target file is in the same folder as the current page, you just need to specify the file name.

For example, from index.html, to link to contact.html:

<a href="contact.html">Contact Us</a>

2. Subfolder

If the target file is in a subfolder, you start the path with the folder name, followed by a slash, and then the file name.

For example, from index.html, to show the image picture.jpg from the images folder:

<img src="images/picture.jpg" alt="A picture">

3. Parent Folder

To link to a file in a folder one level up from the current page, you use two dots (..).

For example, from about.html (inside the pages folder), to link back to index.html:

<a href="../index.html">Go to Homepage</a>

4. Root-relative Path

A path starting with a forward slash (/) is relative to the root of the domain. This can be very useful because it works the same no matter which page you are on.

For example, from any page on the site (like about.html or index.html), you can link to the image like this:

<img src="/images/picture.jpg" alt="A picture">

This tells the browser to start from the root folder of the website and then go into the images folder.

Best Practice

It is best practice to use relative file paths (or root-relative paths) for local links within your own website. This way, your website will not be broken if you change your domain name. It also makes your site more portable, allowing you to move it to a new server without having to update all the paths.

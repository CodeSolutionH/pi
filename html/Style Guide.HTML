HTML Style Guide and Best Practices

A consistent and clean coding style is important for readability and maintainability. This guide outlines some of the most common best practices for writing HTML.

1. Use a Proper Document Structure

Always start your documents with the HTML5 doctype and include the <html>, <head>, and <body> tags. Also, declare the correct language in the lang attribute.

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>Page Title</title>
</head>
<body>
  ...
</body>
</html>

2. Use Lowercase for Tags and Attributes

While HTML is not case-sensitive, it's a strong convention to use lowercase for all element and attribute names. It improves readability.

<p>This is a paragraph.</p>
<a href="#">A link</a>
<P>This is not ideal.</P>
<A HREF="#">This is harder to read.</A>

3. Close All HTML Elements

In HTML5, some elements are self-closing (e.g., <img>, <br>) and some tags are optional to close (e.g., <p>). However, it is a good practice to always close all tags for better readability and to avoid unexpected errors.

<section>
  <p>A paragraph.</p>
  <p>Another paragraph.</p>
</section>

4. Quote Attribute Values

Always enclose attribute values in double quotes. This is the most common convention and prevents errors if the value contains spaces.

<a href="/products/all-items.html">All Items</a>
<a href=/products/all items.html>This is broken</a>

5. Indent Your Code

Indentation makes your code much easier to read by showing the nesting and hierarchy of elements. Use 2 or 4 spaces for each level of indentation.

<ul>
  <li>Coffee</li>
  <li>Tea</li>
</ul>
<ul><li>Coffee</li><li>Tea</li></ul>

6. Use Semantic HTML

Use HTML elements according to their purpose. Semantic HTML (like <header>, <nav>, <article>) improves accessibility, SEO, and code clarity.

<nav>
  <a href="/">Home</a>
</nav>
<div class="navbar">
  <a href="/">Home</a>
</div>

7. Add the alt Attribute to Images

Always include the alt attribute for images. It's crucial for accessibility and is displayed if the image fails to load.

<img src="logo.png" alt="Our company logo">

8. Separate Structure (HTML) from Presentation (CSS)

Avoid using inline styles (the style attribute) whenever possible. Keep your CSS in external stylesheets. This makes the code cleaner and easier to manage.

<!-- In HTML file -->
<link rel="stylesheet" href="styles.css">
<p class="important-note">A note.</p>

<!-- In styles.css file -->
.important-note {
  color: red;
  font-weight: bold;
}
<p style="color:red; font-weight:bold;">A note.</p>

9. Comment Your Code

Use comments to explain complex parts of your code, but don't over-comment simple things. Comments can help you and others understand your code later.

<!-- Main navigation bar for the website -->
<nav>
  ...
</nav>

HTML Semantic Elements

Semantic elements are HTML elements that clearly describe their meaning in a human- and machine-readable way. They accurately describe the purpose of the element and the type of content that is inside them.

Examples of non-semantic elements: <div> and <span>. They tell nothing about their content.

Examples of semantic elements: <form>, <table>, and <article>. They clearly define their content.

Why Use Semantic Elements?

  • Accessibility: They help screen readers and other assistive technologies to understand the page structure and navigate it more easily.
  • SEO (Search Engine Optimization): They provide more information to search engines, which can help them better understand and rank your content.
  • Maintainability: They make the code easier to read and maintain for other developers (and for you in the future).

Code for the Layout Above

    <header>
        <h1>My Website</h1>
    </header>

    <nav>
        <a href="/">Home</a>
        <a href="/about">About</a>
    </nav>

    <section>
        <h2>Main Content Section</h2>
        <article>
            <h3>Blog Post Title</h3>
            <p>Content of the blog post...</p>
        </article>
    </section>

    <aside>
        <h3>Related Links</h3>
        <ul>...</ul>
    </aside>

    <footer>
        <p>Copyright © 2025</p>
    </footer>

Other Semantic Elements

  • <main>: Specifies the main content of a document. The content inside should be unique to the document.
  • <figure> and <figcaption>: Used to group media content (like an image) with a caption.
  • <details> and <summary>: Creates an interactive widget that the user can open and close.
  • <mark>: Defines text that should be marked or highlighted.
  • <time>: Defines a specific time (or a date).

HTML Encoding (Character Sets)

To display an HTML page correctly, a web browser must know the character set (character encoding) used in the file. This ensures that all characters, especially non-English letters and special symbols, are rendered properly.

The <meta charset="..."> Declaration

The character set is specified in a <meta> tag inside the <head> section of your HTML document.

For modern web pages, the UTF-8 character set is strongly recommended. It is the most widely used and supports almost all characters and symbols in the world.

The Correct Syntax

In HTML5, the syntax is simple and should be placed as early as possible within the <head> element:

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>My Web Page</title>
</head>
<body>
...
</body>
</html>

Why is Character Encoding Important?

If a browser misinterprets the character set, it can lead to garbled text, often called "mojibake." For example, a character like the Euro symbol (€) or accented letters (é, ö, ñ) might appear as strange symbols like or Ã©.

Example of Correctly Displayed Characters with UTF-8:

  • Euro: €
  • Copyright: ©
  • Japanese: こんにちは
  • Russian: Здравствуйте
  • Accented: résumé

All these characters are part of the UTF-8 standard.

A Brief History of Character Sets

ASCII

ASCII was the first major character encoding standard. It used 7 bits and contained 128 characters, including numbers (0-9), English letters (A-Z), and some special characters like ! $ + - ( ) @ < >.

ANSI (Windows-1252)

This was the default character set for early versions of Windows. It was an 8-bit character set with 256 characters, adding support for many special characters common in Western European languages.

ISO-8859-1

This was the default character set for HTML 4. It was also an 8-bit set, very similar to Windows-1252. It supported 256 different character codes.

UTF-8 (Unicode)

The limitations of 8-bit character sets (only 256 characters) made it impossible to represent all the world's languages. The Unicode standard was developed to solve this.

UTF-8 is a variable-width character encoding that can represent every character in the Unicode character set. It is backwards compatible with ASCII and has become the de facto standard for the web. It can handle over 1 million character code points, covering nearly all known languages and symbols.

Always save your HTML files with UTF-8 encoding in your text editor to prevent issues. Most modern editors like VS Code, Sublime Text, and Atom do this by default.
